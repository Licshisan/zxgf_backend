# 寻道、旋转延迟与传输时间

## 磁盘 I/O 时间组成

一次传统硬盘访问主要由三部分构成：寻道时间、旋转延迟和传输时间。寻道时间是磁臂移动到目标磁道所需时间；旋转延迟是等待目标扇区转到磁头下方的时间；传输时间是数据真正经过磁头并完成读写的时间。

可以粗略表示为：I/O 时间 = 寻道时间 + 旋转延迟 + 传输时间。这个模型虽然简化，但足以解释为什么小随机 I/O 远慢于大顺序 I/O。

## 寻道时间

寻道涉及磁臂加速、移动、减速和稳定到目标磁道。短距离寻道可能很快，跨盘面大范围移动则更慢。厂商常给出平均寻道时间，但真实时间取决于当前磁头位置和目标位置。

寻道是机械动作，数量级通常是毫秒。对 4KB 随机读写来说，真正传输数据可能只需几十微秒，但寻道和旋转等待会占据绝大多数时间。因此数据库、文件系统和磁盘调度都尽量减少随机寻道。

## 旋转延迟

磁盘以固定速度旋转。若转速为 10000 RPM，则每分钟 10000 圈，每圈约 6ms；平均情况下目标扇区距离磁头半圈，因此平均旋转延迟约 3ms。7200 RPM 磁盘每圈约 8.33ms，平均旋转延迟约 4.17ms。

旋转延迟解释了为什么即使磁头已经在正确磁道，也可能仍需等待。连续读取时，扇区按顺序经过磁头，旋转等待被摊薄；随机读取时，每次请求都可能重新等待半圈。

## 传输时间与带宽

传输时间等于数据大小除以传输速率。对大块顺序 I/O，寻道和旋转只发生一次，随后大部分时间都在传输数据，因此吞吐量接近磁盘峰值带宽。对小块随机 I/O，传输时间极短，整体性能主要被寻道和旋转拖慢。

例如一次 4KB 随机读若花费 8ms，总吞吐只有约 0.5MB/s；一次 100MB 顺序读即使先花 10ms 寻道和旋转，只要后续传输速率 100MB/s，总吞吐仍接近 100MB/s。这种数量级差异是存储系统设计中最重要的性能事实之一。

## 磁盘缓存的影响

磁盘内部通常带有缓存。读时，控制器可能预读同一磁道或相邻扇区，后续顺序读可直接命中缓存；写时，控制器可能先把数据放入缓存再报告完成。缓存能显著改善某些访问模式，但也会改变持久化时机。

如果设备启用写回缓存，系统调用返回成功不一定表示数据已经写到盘面。需要可靠落盘时，文件系统必须使用刷新命令、写屏障或同步接口，确保控制器按要求把缓存内容写入稳定介质。

## 对系统设计的启示

由于机械磁盘随机访问代价高，操作系统和文件系统倾向于顺序化访问：连续分配文件块、合并相邻请求、延迟写入、批量回写、预读和磁盘调度。只要能把大量小随机请求转化为较少的大顺序请求，性能就会明显改善。

但优化不能破坏语义。延迟写和重排请求可能提高吞吐，却会扩大崩溃窗口。存储系统必须在性能和一致性之间明确选择，并用日志、屏障或同步写约束关键顺序。
