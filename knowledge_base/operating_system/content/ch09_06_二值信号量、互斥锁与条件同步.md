# 二值信号量、互斥锁与条件同步

## 二值信号量

二值信号量的值只在 0 和 1 附近变化，可用于互斥。初始值为 1，线程进入临界区前 wait，把值减为 0；其他线程再 wait 会阻塞；退出临界区时 post，把值恢复并唤醒等待者。

这种用法与互斥锁相似，因此常说二值信号量可以当锁使用。

## 与互斥锁的区别

互斥锁通常有持有者概念：谁加锁谁解锁。信号量更像资源计数，post 不一定要求由 wait 的同一线程执行。这个差异在某些同步模式中有用，但也可能让代码更难检查。

互斥锁还可能支持递归属性、优先级继承、错误检查等特性。若只是保护临界区，专用 mutex 通常语义更清楚。

## 条件同步

信号量初始为 0 时可用于等待事件。例如父线程 wait，子线程完成后 post；如果子线程先完成，post 会把值变成 1，父线程之后 wait 也不会阻塞。信号量保存了“事件已发生”的计数。

这种模式适合一次性完成通知、任务计数、资源令牌等场景。若等待条件复杂，如“队列非空且任务属于我”，条件变量通常更合适。

## 有界并发

计数信号量可以限制并发数量。比如连接池有 N 个连接，信号量初始为 N；线程使用连接前 wait，归还连接后 post。这样最多 N 个线程同时持有资源。

这种资源计数语义比用互斥锁加整数手工管理更直接。

## 易错点

二值信号量能实现锁，但不等于所有锁语义。用信号量做条件同步时，要清楚 post 表示资源增加还是事件完成。若把互斥和条件计数混在同一个信号量中，代码很容易出现难以定位的死锁或计数错误。
