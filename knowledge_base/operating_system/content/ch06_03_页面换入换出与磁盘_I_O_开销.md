# 页面换入换出与磁盘 I/O 开销

## 换入和换出

换入是把不在内存中的页面从交换区或文件读入物理页框。换出是把内存中的页面移出，为其他页面腾出页框。若被换出的页是干净页，可以直接丢弃；若是脏页，必须先写回后备存储。

换入通常发生在页错误路径上，对进程延迟影响直接。换出可以在页错误时同步发生，也可以由后台线程提前完成。

## 磁盘 I/O 的数量级

内存访问以纳秒计，磁盘或 SSD 访问远慢于内存。一次页错误若需要存储设备 I/O，开销可能比普通内存访问大数万倍甚至更多。即使命中率只有少量下降，平均访问时间也会被极大拉高。

这就是页面替换策略重要的原因。错误换出很快会再次访问的页，会让系统反复进行慢速 I/O。

## 脏页与干净页

干净页内容与后备存储一致。可执行文件代码页、未修改的文件映射页、已写回的匿名页都可以是干净页。替换干净页成本低，因为无需写盘。

脏页被修改过。替换前必须写回，否则数据丢失。页面守护进程常提前把脏页写回，使未来需要页框时能快速回收干净页。

## I/O 聚集

存储设备通常更擅长顺序大块 I/O。操作系统会把多个换出页面聚集写入，减少寻道、调度和提交开销。即使在 SSD 上，批量提交也能降低软件开销并提高吞吐。

换入也可能预读邻近页面，但过度预读会浪费内存和 I/O。是否预读取决于访问模式是否表现出空间局部性。

## 与调度的关系

发生换入 I/O 的进程会阻塞。调度器可以运行其他就绪进程，从而重叠 CPU 和 I/O。但如果多数进程都在等待换页，CPU 会空闲，系统进入严重内存压力状态。

换页 I/O 也会与普通文件 I/O 竞争设备带宽，影响整个系统响应。

## 易错点

换出页面不是一定要写磁盘，只有脏页必须写回。页错误也不总需要磁盘 I/O，按需置零和写时复制可在内存中完成。真正拖慢系统的是需要慢速后备存储参与的缺页。
