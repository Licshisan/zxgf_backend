# 测试并设置 TAS 与自旋锁

## TAS 指令

测试并设置指令是一种硬件原子原语。它读取某内存位置的旧值，同时把新值写入该位置，并保证这两个动作不可被其他 CPU 或线程打断。常见语义是 `old = *addr; *addr = 1; return old;`，但整个过程原子执行。

没有这种原子性，简单检查锁标志再设置标志会发生竞态：两个线程可能都看到锁空闲，然后都设置为持有，最终同时进入临界区。

## 自旋锁实现

使用 TAS 可以实现自旋锁。锁初始值为 0 表示空闲。线程调用 `TAS(&lock, 1)`，若返回 0，说明它成功把锁从空闲改为占用；若返回 1，说明已有线程持锁，当前线程在循环中继续尝试。释放锁时把标志写回 0。

这种锁在等待时不睡眠，而是反复检查锁状态，所以称为自旋。

## 适用场景

自旋锁适合临界区极短、持锁线程正在其他 CPU 上运行、等待时间预计很小的场景。避免睡眠和唤醒的系统调用开销，是它的主要优势。内核中保护短小数据结构时常使用自旋锁。

在单 CPU 上，若持锁线程被抢占，而等待线程自旋，等待线程会浪费整个时间片，持锁线程反而无法运行释放锁。因此单处理器用户态自旋锁通常表现很差。

## 性能问题

TAS 自旋锁在多核竞争激烈时会造成总线和缓存一致性流量。每个等待线程不断执行原子写，反复使锁所在缓存行失效，拖慢持锁线程和其他等待线程。线程越多，竞争越严重。

改进方法包括先读后 TAS、指数退避、票锁和队列锁。它们减少无意义原子操作或改善公平性。

## 公平性

简单 TAS 锁不保证公平。锁释放时，哪个线程抢到取决于调度和硬件时序，某些线程可能长期失败。对需要严格等待顺序的场景，应考虑票锁或队列锁。

## 易错点

自旋不是无成本等待。它消耗 CPU、内存带宽和缓存一致性资源。TAS 保证锁标志更新原子，但不自动保证临界区代码短，也不避免死锁。持自旋锁时尤其不应执行可能阻塞的操作。
