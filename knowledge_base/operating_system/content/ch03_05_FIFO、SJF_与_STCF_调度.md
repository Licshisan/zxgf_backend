# FIFO、SJF 与 STCF 调度

## FIFO

FIFO 也称 FCFS，按任务到达顺序运行。它的优点是简单、开销低、容易实现；在所有任务运行时间相近且同时到达时，表现也可以接受。队列头部任务运行完成后，调度器再选择下一个任务。

FIFO 的主要问题是护航效应。如果一个长任务排在前面，后面多个短任务即使很快就能完成，也必须等待长任务结束。这样平均周转时间会显著变差。护航效应说明“公平地按到达顺序处理”不一定带来好的性能。

## SJF

SJF 即最短任务优先，总是先运行运行时间最短的任务。在所有任务同时到达、且调度器知道每个任务运行时间、且任务一旦开始就不被抢占的假设下，SJF 可以最小化平均周转时间。

它的直觉很简单：短任务先完成，可以避免大量短任务被少数长任务拖住。超市快速通道就是类似思想。但 SJF 依赖任务长度信息，真实操作系统通常无法准确知道一个进程未来还要运行多久。

## STCF

STCF 即最短完成时间优先，也可理解为抢占式 SJF。每当新任务到达或调度点出现时，调度器比较所有任务的剩余运行时间，选择剩余时间最短的任务。如果新来的短任务比当前任务更快完成，当前任务会被抢占。

STCF 解决了 SJF 在任务陆续到达时的一个问题。假设长任务已经运行，随后来了几个短任务；非抢占式 SJF 只能等长任务完成，而 STCF 可以抢占长任务，让短任务尽快完成，从而降低平均周转时间。

## 共同限制

FIFO 不需要知道运行时间，但容易被长任务拖垮。SJF 和 STCF 在理论上优化周转时间，却依赖对任务长度的先验知识或准确估计。实际系统只能根据历史行为、I/O 模式、优先级或用户提示近似判断。

这些算法主要面向周转时间，不一定有好的响应时间。如果多个任务同时到达，SJF 会让较长任务长时间得不到第一次运行；STCF 也可能不断偏向新来的短任务，使长任务等待过久。

## 与抢占机制的关系

SJF 可以在非抢占式系统中实现；STCF 必须依赖抢占和上下文切换。没有定时器中断和调度点，调度器无法在当前任务尚未结束时切换到新任务。因此调度策略的能力受底层机制约束。

抢占也有成本。频繁抢占会增加上下文切换和缓存失效，因此即使 STCF 在抽象模型中最优，真实系统仍要考虑测量误差和切换开销。

## 易错点

SJF 的“最短”指完整运行时间，STCF 的“最短”指剩余完成时间。FIFO 的问题不是实现错误，而是策略在某些工作负载下不合适。评价这些算法时必须说明假设：任务是否同时到达、是否可抢占、运行时间是否已知、指标是周转时间还是响应时间。
