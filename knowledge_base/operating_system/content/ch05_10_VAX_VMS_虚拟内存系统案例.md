# VAX VMS 虚拟内存系统案例

## 案例价值

VAX/VMS 是理解完整虚拟内存设计的经典案例。它把分段、分页、内核地址空间、页替换和惰性优化组合在一起，展示真实系统如何在硬件限制下做工程取舍。VAX 页大小只有 512 字节，这会放大页表空间和 I/O 效率问题，因此 VMS 必须格外重视页表压缩和换页策略。

这个案例说明，操作系统设计不只是实现教材机制，还要弥补硬件不足、适配不同机器规模，并在性能和通用性之间折中。

## 地址空间结构

VAX 为每个进程提供 32 位虚拟地址空间，并把高位用于区分区域。用户进程空间分为 P0 和 P1：P0 通常放用户代码和向一个方向增长的堆，P1 放向相反方向增长的栈。系统空间 S 位于高地址部分，保存内核代码和数据。

内核空间映射到每个进程地址空间中，但受权限保护。这样系统调用时，内核可以方便访问自己的数据结构，也可以在同一地址空间中复制用户数据，而不必切换到完全独立的内核地址空间。

## 分段加分页

VMS 使用分段加分页的混合方式。P0 和 P1 各有页表，段基址寄存器指向对应页表，界限寄存器记录页表大小。这样，堆和栈之间的大量未使用区域不必占用页表项，缓解小页导致的页表膨胀。

用户页表放在内核虚拟内存中，而不是全部固定占用物理内存。在内存压力下，页表页本身也可以被换出。这进一步降低了页表常驻内存压力，但让地址转换依赖 TLB 的程度更高。

## 保护与零页

VMS 的地址空间设计中，低地址零页通常标为不可访问，用来捕获空指针引用。程序解引用空指针时产生虚拟地址 0，硬件查表发现无效，陷入内核并报告错误。

内核页设置更高保护级别，用户态不能读写。保护字段让同一地址空间中既包含用户区域又包含内核区域，同时保持安全边界。

## 页替换与二次机会

VMS 面对没有硬件引用位的问题，采用了分段 FIFO 和二次机会列表。每个进程有驻留集大小限制，超过限制时按 FIFO 选择页移出。被移出的页不会立刻彻底丢弃，而是进入全局干净页列表或脏页列表。

如果原进程很快再次访问该页，仍可从二次机会列表取回，避免磁盘 I/O。脏页可以批量写回，使之后回收更便宜。这种设计在缺少精确 LRU 支持时提供了实用近似。

## 惰性优化

VMS 使用按需置零和写时复制。按需置零在扩展地址空间时先不立刻分配并清零物理页，只在首次访问时触发异常，再分配零页。写时复制在复制地址空间时先共享只读页，只有写入时才真正复制。

这些惰性技术减少不必要工作。如果页面从未访问，按需置零避免分配；如果 fork 后马上 exec，写时复制避免复制大量无用页面。

## 易错点

VMS 案例不是单一机制，而是一组机制的组合。小页大小使页表和 I/O 问题更突出；内核映射进每个地址空间不等于用户可访问内核；二次机会列表不是严格 LRU，而是在硬件限制下的实用近似。
