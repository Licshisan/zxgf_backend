# 动态重定位与基址界限寄存器

## 基本思想

动态重定位是一种早期而重要的地址转换机制，也称基址加界限机制。每个运行进程拥有一对硬件寄存器：基址寄存器记录该进程地址空间在物理内存中的起始位置，界限寄存器记录地址空间大小或允许访问范围。用户程序产生虚拟地址后，硬件用基址加上虚拟地址得到物理地址，并用界限检查访问是否合法。

例如进程认为自己访问虚拟地址 1000，基址寄存器为 32000，则实际访问物理地址 33000。若虚拟地址大于等于界限，硬件触发异常，内核通常终止该进程或报告内存错误。

## 为什么是动态

静态重定位由加载器在装入程序时修改指令中的地址，把它们改成实际物理地址。这种方法缺少灵活性，也难以提供保护。动态重定位在程序运行时由硬件完成转换，程序仍然使用从 0 开始的虚拟地址，进程放到物理内存哪里由操作系统决定。

运行时转换还有一个好处：上下文切换时，只要保存和恢复基址、界限寄存器，就能让同一套虚拟地址在不同进程中映射到不同物理区域。

## 硬件支持

基址界限机制需要处理器模式、基址寄存器、界限寄存器、地址加法和越界检查电路。修改这些寄存器的指令必须是特权指令，只能由内核执行。否则用户进程可以把基址改到内核区域或其他进程区域，保护机制就失效了。

MMU 在每次取指、读数据和写数据时都执行转换。这个路径必须非常快，因为内存访问频率极高。基址加界限的硬件逻辑简单，因此效率好，是理解更复杂地址转换机制的起点。

## 操作系统职责

进程创建时，操作系统要在物理内存中找到一块足够大的连续空间，装入进程地址空间，并设置该进程的基址和界限。进程退出时，内核释放这块物理空间。上下文切换时，内核把当前进程的寄存器值保存到 PCB，再恢复下一个进程的基址和界限。

如果进程发生越界访问，硬件陷入内核。内核需要判断异常原因，并采取终止进程、发送信号或其他错误处理。对普通用户进程来说，这类错误常表现为段错误或访问违规。

## 局限

基址界限要求整个地址空间连续放入物理内存。若进程地址空间很大但实际只用少量区域，仍可能浪费大量物理内存。若物理内存中没有足够大的连续空闲块，即使总空闲空间够，也无法装入进程。

它也不支持细粒度共享。代码段、堆、栈被当作一个整体搬动和保护，无法方便地只共享只读代码或只给某段设置写保护。分段和分页正是为了解决这些问题而发展。

## 易错点

基址寄存器保存的是物理起点，不是虚拟起点；进程看到的虚拟地址仍从 0 或约定位置开始。界限检查保护的是进程地址空间边界，不会自动理解代码、堆、栈的内部语义。它简单高效，但灵活性有限。
