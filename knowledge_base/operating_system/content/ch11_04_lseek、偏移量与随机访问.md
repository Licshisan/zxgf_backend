# lseek、偏移量与随机访问

## 当前文件偏移量

每个普通打开文件对象通常维护一个当前文件偏移量。顺序读写时，`read` 或 `write` 从当前偏移开始，成功处理 N 个字节后，偏移量自动增加 N。这个状态属于打开文件对象，而不是文件本身，也不只是描述符整数。

因此，两个独立 `open` 得到的打开对象有各自偏移量；`dup` 或 `fork` 继承后共享同一个打开对象时，偏移量也会共享。这个细节会影响并发读写和 shell 重定向行为。

## lseek 的作用

`lseek(fd, offset, whence)` 只改变内核记录的当前文件偏移量。`SEEK_SET` 表示把偏移设为给定值，`SEEK_CUR` 表示相对当前位置移动，`SEEK_END` 表示相对文件末尾移动。调用成功后，下一次读写从新位置开始。

`lseek` 本身不会读写磁盘，也不会直接移动磁盘磁头。它只是修改内存中的打开文件状态。之后如果程序从相距很远的位置读写，底层磁盘访问可能产生实际寻道，但那是后续 I/O 的结果。

## 随机访问

随机访问允许程序读取或更新文件中的任意位置。数据库、索引文件、虚拟磁盘镜像和大型日志都依赖这种能力。没有随机访问，程序只能从头到尾扫描，许多数据结构无法高效实现。

随机访问也可能损害机械磁盘性能。连续访问可以摊薄寻道和旋转延迟，随机小块访问则频繁等待机械动作。文件系统缓存、预读、请求合并和块分配策略都试图改善这种代价。

## 文件空洞

如果 `lseek` 把偏移移动到文件末尾之后，再执行写入，中间未写区域可形成文件空洞。许多文件系统不会为这些全零区域分配实际磁盘块，读取空洞时返回零。这类稀疏文件常用于虚拟机镜像、数据库文件和大型占位文件。

空洞说明文件大小和实际占用块数不是一回事。应用看见的是逻辑字节范围，文件系统内部可以用元数据表示未分配区域，以节省空间。

## pread 与 pwrite

多线程程序若共享打开文件对象，使用 `lseek` 加 `read` 的组合可能产生竞态：一个线程刚设置偏移，另一个线程就改掉它。`pread` 和 `pwrite` 接收显式偏移，不改变当前文件偏移量，因此更适合并发随机访问。

这体现了接口设计中的状态问题。隐式偏移让顺序读写简单，但共享状态会带来并发复杂度；显式偏移调用更冗长，却能避免许多同步问题。
