# 磁盘结构：磁道、扇区与柱面

## 磁盘的外部接口

从操作系统角度看，传统硬盘通常表现为一段线性块地址空间。磁盘包含编号从 0 到 n-1 的扇区，每个扇区可读可写。经典扇区大小是 512 字节，现代磁盘也常见 4KB 物理扇区。文件系统一般以更大的块为单位工作，例如 4KB 或更大。

这个接口故意隐藏了真实几何结构。操作系统提交逻辑块号，磁盘控制器负责映射到具体盘片、表面、磁道和扇区。即便如此，几何结构仍然影响性能，因为相邻逻辑块通常更容易连续访问，随机块则可能引起大量机械移动。

## 盘片、表面与主轴

硬盘由一个或多个盘片组成，盘片表面涂有磁性材料，用磁化状态保存数据。每个盘片通常有两个可记录表面，所有盘片固定在同一个主轴上，由电机以恒定速度旋转。转速以 RPM 表示，例如 7200 RPM 表示每分钟旋转 7200 圈。

每个表面都有对应磁头。磁头安装在同一个磁臂组件上，因此多个磁头通常一起移动到相同半径位置。读写时，磁头不直接接触盘面，而是在极小距离上感应或改变磁性状态。

## 磁道、扇区与柱面

磁道是盘面上一圈同心圆轨迹，数据沿磁道分成多个扇区。扇区是磁盘可寻址的最小传统单位，包含用户数据以及校验、同步等附加信息。相同半径上、不同盘面的一组磁道称为柱面。早期系统常用柱面概念优化访问，因为切换同一柱面内的磁头比移动磁臂更快。

现代磁盘内部映射更复杂，操作系统通常不再直接管理柱面、磁头和扇区编号，但这些概念仍有助于理解为什么顺序访问快、随机访问慢，以及为什么磁盘调度算法关注磁臂移动。

## 多区域记录与磁道偏斜

外圈磁道周长更长，可容纳更多扇区。现代磁盘常采用多区域记录，把相邻若干磁道划为一个区域，同一区域内每道扇区数相同，外圈区域扇区数多于内圈区域。这意味着磁盘不同位置的顺序传输速率可能不同。

磁道偏斜用于改善跨磁道顺序访问。磁头从一个磁道移动到相邻磁道需要时间，如果两个磁道的逻辑连续扇区完全对齐，移动完成时目标扇区可能刚刚错过磁头，必须等一整圈。通过让相邻磁道的扇区编号错开，磁盘可以在切换磁道后更快接上后续数据。

## 原子性与不完整写入

磁盘通常只保证单个扇区写入具备原子性，即要么完整写入，要么不写入。更大的多扇区写入在断电或故障时可能只完成一部分，形成不完整写入。文件系统日志、写屏障和校验机制正是为了处理这种现实。

因此不能把“写入 4KB 文件系统块”简单等同于“磁盘已经原子更新 4KB”。当物理扇区、文件系统块、磁盘缓存和控制器重排同时存在时，持久化语义必须由文件系统和设备协议共同保证。
