# 内存虚拟化：影子页表与嵌套页表

## 三层地址

普通操作系统把进程虚拟地址转换为物理地址。虚拟化后，客户操作系统看到的“物理地址”并不是真实机器地址，而是客户物理地址。VMM 还要把客户物理地址映射到真实机器地址。

因此存在三层：客户进程虚拟地址、客户物理地址、机器地址。VMM 必须让客户 OS 以为自己管理物理内存，同时阻止它访问未分配给自己的机器内存。

## 影子页表

影子页表由 VMM 维护，直接把客户虚拟地址映射到机器地址。硬件实际使用影子页表进行转换，客户 OS 维护的页表则被 VMM 监控。客户 OS 修改页表时，VMM 更新对应影子页表。

影子页表能利用现有硬件 TLB，但维护成本高。客户 OS 的页表写入、上下文切换、权限变化都可能导致影子页表同步、失效或重建。

## 写保护与同步

为了发现客户 OS 修改页表，VMM 可以把客户页表页设为只读。客户写页表时触发陷入，VMM 检查修改是否合法，再更新影子页表。这保证控制权，但频繁页表操作会带来大量 VM exit。

TLB 一致性也更复杂。客户以为自己刷新的是客户地址转换，VMM 还要确保真实硬件不继续使用过期的影子映射。

## 嵌套页表

硬件辅助虚拟化引入嵌套页表，例如 Intel EPT 或 AMD NPT。客户 OS 管理客户虚拟到客户物理的页表，VMM 管理客户物理到机器地址的页表。硬件在地址转换时自动走两级页表。

嵌套页表减少了影子页表同步开销。客户 OS 修改自己的页表通常不必陷入 VMM。但一次 TLB 未命中可能需要更多内存访问，因为硬件要遍历两套页表。

## 内存过量分配

VMM 还可以在虚拟机之间做内存复用。常见技术包括气球驱动、页共享、换出虚拟机内存和内存压缩。过量分配能提高资源利用率，但会增加不可预测延迟。

内存虚拟化的核心取舍是透明性、隔离和转换成本。影子页表偏软件维护，嵌套页表偏硬件支持，二者都围绕同一目标：让客户系统安全地共享真实机器内存。
