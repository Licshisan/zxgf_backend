# 近似 LRU 与时钟算法

## 为什么需要近似

精确 LRU 需要记录每次访问的全局顺序，成本太高。操作系统更常使用硬件引用位来近似近期性。页面被访问时，硬件设置引用位；内核周期性检查并清除它，从而判断页面最近是否用过。

近似算法牺牲精确性，换取低开销和可实现性。只要大致区分热页和冷页，就能显著好于 FIFO。

## 时钟算法

时钟算法把候选页面组织成环形队列，用一个指针像钟表指针一样扫描。需要淘汰页面时，检查指针指向页面的引用位。若引用位为 1，说明近期访问过，清零并给第二次机会，指针前进。若引用位为 0，则选择该页淘汰。

这种方法也称 Second Chance。它近似 LRU：近期访问过的页面会被跳过，长时间未访问的页面最终引用位保持 0，被选中。

## 脏页处理

时钟算法常结合脏位。干净且未引用的页面最适合淘汰；脏且未引用的页面需要写回，成本更高；被引用的页面暂时保留。有些系统优先清理脏页或把脏页放入写回队列，之后再作为干净页回收。

这让替换策略同时考虑未来命中概率和回收成本。

## 多轮扫描

如果所有页面引用位都为 1，时钟指针会清除一圈引用位，第二圈才能找到候选页。这说明系统工作集可能很活跃。扫描成本通常仍低于维护精确 LRU，但在内存压力很高时也会增加 CPU 开销。

## 变体

增强型 Clock、老化算法和 WSClock 会记录更多历史或结合工作集信息。老化算法周期性右移计数器并把引用位放入高位，使计数器近似表示近期访问频率。WSClock 同时考虑页面年龄、引用位和脏位。

## 易错点

引用位为 0 不表示页面从未访问，只表示自上次清除后未访问。时钟算法不是 FIFO，虽然也有环形队列；它会根据引用位给页面第二次机会。近似 LRU 的目标是低成本接近，而不是完全复制 LRU 顺序。
