# 禁用中断与单处理器同步

## 基本思想

在单处理器系统中，线程切换通常依赖中断或系统调用。如果内核在进入临界区前禁用中断，就能阻止当前 CPU 被中断打断，从而让临界区连续执行。退出临界区后再恢复中断。

这种方法曾是早期互斥方案，也仍在某些内核短路径中使用。它利用的是单 CPU 上“没有中断就没有抢占”的事实。

## 适用范围

禁用中断只适合内核内部非常短的临界区，且必须由可信代码使用。内核可以控制持有时间，确保不会长期关闭中断，避免影响设备响应和系统时钟。

用户程序不应拥有随意关闭中断的能力。若用户线程能关闭中断，它可以独占 CPU，甚至让系统无法响应设备和调度。

## 多处理器失效

禁用当前 CPU 的中断，不能阻止其他 CPU 上的线程同时运行。如果两个 CPU 同时访问共享数据，只关闭本地中断无法提供互斥。多处理器内核需要自旋锁、原子指令和跨核同步。

因此禁用中断通常只用于保护会被本 CPU 中断处理程序同时访问的数据，或与自旋锁配合避免中断上下文死锁。

## 风险

关闭中断会延迟定时器、设备完成通知和网络包处理。持有时间过长会造成时钟漂移、I/O 延迟和系统响应变差。若代码在关闭中断后出错、阻塞或进入长循环，系统可能严重卡死。

因此内核必须严格限制关闭中断区间，只执行确定、短小、不会睡眠的操作。

## 与锁的关系

禁用中断是一种特殊同步机制，不是通用锁替代品。普通互斥锁适合线程之间同步；自旋锁适合短临界区；禁用中断适合阻止本 CPU 上中断处理程序重入。不同上下文需要不同组合。

## 易错点

单处理器上的禁用中断能防抢占，但不能保护多处理器共享数据。禁用中断也不能在用户态作为同步工具。它的正确性依赖代码可信、临界区短且不会阻塞。
