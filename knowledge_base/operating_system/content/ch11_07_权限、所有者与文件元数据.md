# 权限、所有者与文件元数据

## 文件元数据

文件系统保存的不只是文件内容，还包括元数据。常见元数据有文件类型、大小、所有者、所属组、权限位、时间戳、链接计数以及指向数据块的索引信息。UNIX 中这些信息主要保存在 inode 中。

元数据决定文件如何被解释和访问。目录、普通文件、符号链接、设备文件和管道可能都出现在目录树里，但它们的类型位不同，内核对系统调用的处理方式也不同。

## 所有者与组

每个文件通常有用户 ID 和组 ID。权限检查会根据调用进程的有效用户 ID、有效组 ID 和附加组列表判断是否允许访问。所有者可以控制文件权限，超级用户或具备相应能力的进程可以绕过部分限制。

创建文件时，所有者一般是创建进程的有效用户，组可能来自进程有效组或父目录策略。目录上的 setgid 位常用于让新文件继承目录组，便于团队共享目录管理。

## 权限位

传统 UNIX 权限分为所有者、组、其他人三类，每类有读、写、执行三个位。对普通文件，读表示可读取内容，写表示可修改内容，执行表示可作为程序执行。对目录，读表示可列出目录项，写表示可增删改目录项，执行表示可进入或穿过目录。

目录权限容易被误解。只有目录读权限但没有执行权限，通常无法访问其中具体条目；有执行权限但无读权限，可能无法列目录，但若知道文件名仍可访问。删除文件取决于所在目录权限，而不主要取决于文件本身写权限。

## 时间戳与状态变化

常见时间戳包括访问时间、修改时间和状态改变时间。修改时间反映文件内容变化，状态改变时间反映元数据变化，如权限、所有者或链接计数改变。访问时间记录读取行为，但出于性能考虑，现代系统可能延迟更新或采用相对宽松策略。

这些时间戳被构建工具、备份程序、缓存系统和审计工具使用。若文件系统缓存或挂载选项改变更新时间语义，相关程序行为也会受到影响。

## umask、chmod 与 chown

`umask` 限制新建文件的默认权限。程序传给 `open` 或 `mkdir` 的模式会被 umask 屏蔽，避免默认创建过于开放的文件。`chmod` 修改权限位，`chown` 修改所有者或组，通常受权限限制。

权限和元数据属于文件系统安全边界的一部分。正确理解它们，需要同时考虑路径中每一级目录权限、目标文件权限、进程身份以及特殊位，而不是只看最终文件的读写位。
