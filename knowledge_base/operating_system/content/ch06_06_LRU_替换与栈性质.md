# LRU 替换与栈性质

## LRU 的思想

LRU 即最近最少使用，淘汰最长时间没有被访问的页面。它基于时间局部性：最近访问过的页面更可能再次访问，长时间没访问的页面相对更适合淘汰。

相比 FIFO，LRU 关注使用历史而非进入内存时间。一个很早进入但持续被访问的热点页，在 LRU 下会被保留。

## 精确 LRU 的成本

精确 LRU 需要在每次页面访问时更新顺序，或者记录精确时间戳。内存访问极其频繁，若每次都让操作系统更新链表，开销无法接受。硬件可以提供引用位或时间信息，但完整精确 LRU 仍然昂贵。

因此真实系统通常实现近似 LRU。它们用引用位、Clock 算法、老化计数等方法近似判断近期使用情况。

## 栈性质

LRU 具有栈性质：容量为 N 的缓存内容，总是容量为 N+1 的缓存内容的子集。直观上，更大的 LRU 缓存不会比更小缓存少保存近期访问集合。因此增加缓存容量不会导致命中率下降。

FIFO 不具备这个性质，所以可能出现 Belady 异常。栈性质是理解 LRU 稳定性的关键。

## LRU 的表现

在具有良好时间局部性的工作负载中，LRU 表现接近最优。例如循环反复访问一个小工作集时，工作集进入内存后会持续命中。若访问模式是一次性顺序扫描，LRU 可能把热点页挤掉，表现并不理想。

LRU 也会被循环工作集大小略大于内存容量的模式击败。每次访问都可能淘汰下一轮马上需要的页。

## 频率与近期性的区别

LRU 看近期，不看总频率。一个页面历史上访问很多次，但最近不用，可能被淘汰；LFU 看频率，可能保留过去热点但现在无用的页面。实际策略常结合近期和频率，或使用衰减避免旧历史长期影响。

## 易错点

LRU 不是最优策略，只是实用近似。精确 LRU 和近似 LRU 也不是一回事。栈性质说明容量增加不会让 LRU 变差，但不说明 LRU 在所有访问序列上都接近最优。
