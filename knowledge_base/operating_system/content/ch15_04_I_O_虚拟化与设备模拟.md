# I/O 虚拟化与设备模拟

## I/O 为什么难

CPU 和内存虚拟化主要处理指令和地址转换，I/O 虚拟化还要面对大量异构设备、寄存器协议、中断、DMA 和性能要求。客户 OS 期望操作磁盘、网卡和显示设备，但不能直接控制真实硬件。

VMM 必须在透明性、性能和隔离之间平衡。完全模拟真实设备兼容性好，但开销大；让虚拟机直接访问设备性能高，但隔离和共享更困难。

## 设备模拟

设备模拟让 VMM 呈现一个常见硬件设备，例如虚拟 IDE 磁盘或 E1000 网卡。客户 OS 使用已有驱动访问该设备，读写寄存器或提交命令时陷入 VMM，由 VMM 模拟设备行为。

这种方式不需要修改客户 OS，适合启动和兼容。但每次寄存器访问、PIO 或中断都可能带来陷入开销，吞吐和延迟不如专门虚拟化接口。

## 半虚拟化设备

半虚拟化设备让客户 OS 使用专门驱动，直接与 VMM 设计的高效接口交互。例如 virtio 通过共享队列提交请求，减少复杂设备模拟。客户驱动知道自己运行在虚拟化环境中，但接口更简单高效。

半虚拟化牺牲部分透明性，换取性能和实现简洁。云环境和现代虚拟机中，这类设备非常常见。

## DMA 与 IOMMU

真实设备使用 DMA 直接读写内存。若把设备直通给虚拟机，必须防止设备访问其他虚拟机或 VMM 的内存。IOMMU 提供设备侧地址转换和权限控制，让 DMA 地址也受到隔离。

设备直通性能接近裸机，但共享和迁移困难。一个物理设备被绑定给某个虚拟机后，其他虚拟机不能轻易同时使用。SR-IOV 等技术通过虚拟功能改善多租户共享。

## 中断虚拟化

设备完成请求后产生中断。VMM 要把真实中断转换为对应虚拟机的虚拟中断，或者在虚拟设备队列中标记完成。频繁中断会造成大量虚拟机进出，因此常用中断合并和批处理。

I/O 虚拟化的性能常由数据路径决定。高效方案会尽量减少拷贝、减少陷入、批量提交请求，并用共享内存队列连接客户和 VMM。
