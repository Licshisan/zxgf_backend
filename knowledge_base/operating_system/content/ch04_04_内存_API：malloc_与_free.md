# 内存 API：malloc 与 free

## 栈内存与堆内存

C 程序常见内存来源有栈和堆。栈内存由编译器和运行时自动管理，进入函数时为局部变量和调用帧分配空间，函数返回时释放。它生命周期短，适合保存临时局部状态。

堆内存由程序员显式申请和释放。`malloc()` 从堆中分配指定字节数，返回指向可用区域的指针；`free()` 把之前分配的区域归还给分配器。堆适合保存生命周期跨越函数调用的数据结构，如链表、树、缓存和动态数组。

## malloc 的使用

`malloc(size)` 的参数是需要的字节数，成功返回 `void *` 指针，失败返回 `NULL`。实际代码中应使用 `sizeof` 计算类型大小，例如：

```c
int *a = malloc(n * sizeof(int));
```

这样比手写字节数更可靠。分配字符串时要给结尾的 `'\0'` 留空间，常见写法是 `malloc(strlen(s) + 1)`。如果使用 `sizeof(pointer)` 误以为得到动态数组大小，就会出错，因为它只返回指针本身的大小。

## free 的语义

`free(ptr)` 释放由 `malloc()`、`calloc()`、`realloc()` 等分配函数返回的指针。调用者不需要传入大小，因为分配器会在内部记录块大小，常见方式是在用户可见指针前保存头部元数据。

释放后，指针变量本身的值不会自动改变，但它指向的内存已经不再属于程序。继续通过该指针读写就是悬空指针错误。为降低误用风险，释放后把指针置为 `NULL` 是常见习惯，但这只保护当前变量，不能消除其他别名指针。

## calloc 与 realloc

`calloc(count, size)` 分配数组并把内容初始化为 0，适合需要零初始化的结构。它还可以在实现中检查乘法溢出，安全性通常优于手写 `malloc(count * size)` 后再清零。

`realloc(ptr, new_size)` 调整已有块大小。它可能原地扩展，也可能分配新块、复制旧内容并释放旧块。若返回新指针，旧指针不能继续使用。处理 `realloc` 失败时要小心，直接把返回值覆盖原指针可能导致原块丢失而泄漏。

## 与操作系统的关系

`malloc()` 通常是用户态库函数，不是每次都直接进入内核。分配器会管理进程堆中的空闲块；当堆空间不足时，才通过 `sbrk()`、`mmap()` 等系统调用向内核申请更多虚拟内存。`free()` 也不一定立刻把内存还给操作系统，可能只是放回用户态分配器的空闲列表。

这解释了为什么程序释放内存后，进程占用的虚拟内存或常驻内存不一定马上下降。

## 易错点

`malloc()` 返回的内存未初始化，读取前必须写入。`free()` 只能释放分配器返回的起始指针，不能释放栈地址、全局变量地址或块中间地址。每块动态内存应有清晰所有权：谁负责释放、何时释放、释放后谁不能再访问，都必须明确。
