# RAID 基本思想：条带化、镜像与校验

## RAID 要解决的问题

单块磁盘容量有限、性能有限，并且一旦故障可能丢失数据。RAID 使用多块磁盘组成一个对主机透明的逻辑磁盘，目标是提高容量、吞吐量和可靠性。文件系统看到的仍是一段线性块地址空间，RAID 控制器或软件 RAID 负责把逻辑块映射到多个物理磁盘。

RAID 的关键价值在于透明性。主机不必为每块物理磁盘单独设计文件系统，管理员可以用阵列替代单盘，获得更大的空间、更高的并行度或一定容错能力。

## 条带化

条带化把连续逻辑块分散到不同磁盘。例如四块盘中，逻辑块 0、1、2、3 分别放到四块盘，随后 4、5、6、7 再分布一轮。这样大顺序读写可以并行使用多块磁盘，提高带宽。

条带化的基本参数是 chunk size，即每块磁盘上连续放置的数据单元大小。较小 chunk size 让单个文件更容易跨盘并行，但可能让一次请求涉及更多磁盘、增加协调开销；较大 chunk size 更依赖多个并发请求利用并行性。最佳大小取决于工作负载。

## 镜像

镜像为每个数据块保存两个或更多副本，副本放在不同磁盘上。读请求可以从任一副本读取，控制器可选择空闲盘或距离更近的位置；写请求必须更新所有副本，才能保证后续读到一致数据。

镜像的可靠性强，实现简单，故障恢复也直接：坏盘替换后，从健康副本复制数据即可。代价是容量利用率低，双副本镜像只能提供一半可用容量。对于重视可靠性和随机读性能的系统，镜像仍然非常常见。

## 校验

校验通过额外信息恢复丢失数据，最常见的是异或校验。若一组数据块 D0、D1、D2 和校验块 P 满足 P = D0 xor D1 xor D2，则任意一块丢失时，都可用其余块异或恢复。例如 D1 丢失时，D1 = P xor D0 xor D2。

校验比镜像节省空间，N 块磁盘中通常只牺牲相当于一块磁盘的容量即可容忍一个磁盘故障。但写入更复杂，特别是小写入需要读取旧数据和旧校验，计算新校验，再写回数据和校验。

## 容量、可靠性与性能

评价 RAID 不能只问“哪种最好”，必须同时看容量、可靠性和性能。条带化容量和性能好，但没有冗余；镜像可靠且读性能好，但容量成本高；校验阵列容量效率高，顺序性能好，但小随机写通常较慢。

可靠性也要结合故障模型。简单模型假设整盘故障可被立即检测，现实还包括潜在扇区错误、无声数据损坏、控制器故障和重建期间第二块盘故障。RAID 提高可用性，但不能替代备份。
