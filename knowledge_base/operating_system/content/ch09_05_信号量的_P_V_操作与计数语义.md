# 信号量的 P V 操作与计数语义

## 信号量是什么

信号量是带整数计数的同步对象。经典操作称为 P 和 V，在 POSIX 中分别近似为 `sem_wait()` 和 `sem_post()`。P 操作减少计数，若结果表示资源不足则等待；V 操作增加计数，并在有等待者时唤醒一个线程。

信号量把“计数”和“等待队列”结合在一个原语中，既可表达互斥，也可表达资源数量。

## 计数语义

信号量值大于 0 时，表示还有可立即获取的资源数量。线程执行 wait 会消耗一个资源并继续。值为 0 时，资源刚好用完，后续 wait 需要等待。在某些定义中，值为负数时，其绝对值表示等待线程数量。

post 不等待条件，它只是归还或增加资源，并唤醒等待者之一。

## 初始值的重要性

信号量行为由初始值决定。初始为 1 时，可作为互斥锁；初始为 N 时，可限制最多 N 个线程同时进入某区域；初始为 0 时，可用于事件等待，表示事件尚未发生，等待者会阻塞直到其他线程 post。

错误初始值会导致互斥失效或线程永久等待。

## 原子性

wait 和 post 必须是原子的。多个线程同时修改信号量值和等待队列时，不能出现丢失唤醒或计数错误。信号量内部实现通常需要锁、条件变量或硬件原子指令，但使用者看到的是不可分割操作。

## 与条件变量的区别

条件变量不保存事件，signal 时没有等待者通常就丢失；信号量保存计数，post 可以发生在 wait 之前，之后 wait 会直接消耗计数继续。因此信号量适合资源计数，条件变量适合等待复杂谓词。

## 易错点

信号量值不是普通共享变量，不应在应用层随意读取并据此判断是否等待。P/V 顺序必须与资源获取释放一致。用信号量可以写出正确同步，也可以写出非常难读的代码；复杂条件往往用锁加条件变量更清晰。
