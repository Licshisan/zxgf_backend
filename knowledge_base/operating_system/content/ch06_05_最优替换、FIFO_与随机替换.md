# 最优替换、FIFO 与随机替换

## 最优替换

最优替换策略选择未来最久才会再次访问的页面淘汰。它能产生最低缺页次数，是理论比较基准。但操作系统无法知道未来访问序列，因此不能在真实系统中直接实现。

最优策略的价值在于评估。若某个实际策略已经接近最优，继续优化空间有限；若差距很大，说明策略或负载处理还有问题。

## FIFO 替换

FIFO 按页面进入内存的时间排序，淘汰最早进入的页面。它实现简单，只需维护队列。页面换入时加入队尾，替换时从队头取出。

FIFO 的问题是完全不考虑页面是否最近使用或频繁使用。一个很热的页面只要进入时间早，也会被淘汰。它还可能出现 Belady 异常：增加缓存容量反而导致更多缺页，这是缺少栈性质的表现。

## 随机替换

随机替换在需要淘汰时随机选择一个页面。它实现简单，避免某些固定顺序的最坏模式，也不需要维护复杂历史信息。多次运行的表现取决于随机结果，长期平均可能还可以。

随机策略的问题是不可预测，可能淘汰重要页面。它常作为简单基线或在某些硬件缓存中使用，但通用操作系统通常会利用更多历史信息。

## 冷启动缺页

缓存开始为空时，第一次访问某页面一定缺页，这称为冷启动缺页或强制缺页。替换策略无法避免这类缺页。评价策略时，有时会把冷启动缺页单独列出，重点比较容量不足时的替换效果。

## 策略比较

最优策略利用未来，无法实现但最好；FIFO 利用到达顺序，实现简单但可能很差；随机不利用历史，实现也简单，避免部分极端队列行为但缺乏稳定保证。它们共同说明：替换策略必须在信息量、复杂度和性能之间取舍。

## 易错点

FIFO 的“先进先出”不是公平性的保证，它只是按进入内存时间淘汰。随机替换不是没有逻辑，而是用低状态成本换取平均表现。最优替换不可实现，不应被当作实际算法背诵，而应作为上限基准理解。
