# 脏页写回与后台分页守护进程

## 脏页为什么麻烦

脏页包含尚未写回后备存储的修改。若替换时才发现页面是脏的，当前缺页处理必须等待写盘，再等待目标页读入，延迟很高。干净页则可直接回收，成本低得多。

因此操作系统会主动管理脏页，让一部分页面在真正需要回收前已经变干净。

## 后台分页守护进程

多数系统设置低水位线和高水位线。当空闲页数低于低水位线，后台分页线程开始扫描页面，选择牺牲页并回收，直到空闲页数达到高水位线。它常被称为 page daemon 或 swap daemon。

后台处理让前台缺页不必总是临时找页、写回和等待。只要空闲页池保持充足，页错误可以更快获得页框。

## 写回策略

后台线程会把脏页批量写回交换区或文件，使其转为干净页。批量写回可以合并相邻 I/O，提高设备效率。写回完成后，页面可留在内存中作为干净缓存，也可以被放入可回收列表。

写回不能过于激进。过早写回会增加无用 I/O，因为页面可能很快再次被修改；过晚写回则在内存压力来临时造成前台等待。

## 与文件系统缓存

文件页和匿名页的写回路径不同。文件页写回对应文件系统，必须遵守文件一致性和 `fsync()` 语义；匿名页写入交换空间。二者都参与内存回收，但后备存储和持久性语义不同。

干净文件页常可直接丢弃，因为需要时从文件重读即可。匿名页没有原始文件，若内容有用，必须保存在交换区。

## 水位线的意义

水位线让系统提前应对内存压力，而不是等完全无页可用才行动。高水位线和低水位线之间形成缓冲，避免后台线程频繁启动停止。水位设置过低会让前台频繁等待；过高则可能过度回收，影响缓存命中。

## 易错点

写回脏页不等于释放页面。写回只是让页面变干净，是否释放取决于回收策略。后台分页守护进程也不是只在系统崩溃前运行，它是正常内存管理的一部分。
