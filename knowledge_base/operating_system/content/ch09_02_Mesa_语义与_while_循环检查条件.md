# Mesa 语义与 while 循环检查条件

## Hoare 语义与 Mesa 语义

条件变量有不同语义模型。Hoare 语义中，signal 后被唤醒线程立即获得执行权，条件在它运行时仍然成立。Mesa 语义中，signal 只是把等待线程从睡眠变为就绪，唤醒线程要等调度器安排，并且重新竞争锁。

POSIX 条件变量采用 Mesa 风格。因此，线程被唤醒时，通知方或其他线程可能已经改变了条件。

## 为什么必须用 while

等待条件时应使用 `while`，而不是 `if`。`if` 只在睡眠前检查一次；线程醒来后会直接继续执行。Mesa 语义下，醒来只说明“可能有变化”，不保证条件成立。

使用 `while` 可以在每次醒来并重新获得锁后再次检查条件。若条件仍不满足，线程继续 wait。这是条件变量正确使用的基本模板。

## 虚假唤醒

一些实现允许虚假唤醒，即线程可能在没有明确 signal 的情况下从 wait 返回。即使不考虑虚假唤醒，多个等待者被唤醒后也可能只有一个真正获得资源。`while` 循环能统一处理这些情况。

因此不要写依赖“signal 必然对应一次成功消费”的代码。条件变量通知的是状态变化，不是资源所有权转移。

## 生产者消费者中的例子

消费者等待缓冲区非空。若用 `if (count == 0) wait()`，多个消费者被唤醒后，第一个消费者取走数据，第二个消费者醒来后可能继续从空缓冲区取数据。用 `while (count == 0) wait()`，第二个消费者会重新睡眠。

生产者等待缓冲区非满也同理。条件检查必须放在持锁状态下，与修改 `count` 的代码使用同一把锁。

## 易错点

signal 不会把锁直接交给被唤醒线程。被唤醒线程只是变为可运行，返回 wait 前仍需重新获得锁。`while` 不是保守写法，而是 Mesa 条件变量语义下的正确写法。
