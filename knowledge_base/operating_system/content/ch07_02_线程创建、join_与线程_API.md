# 线程创建、join 与线程 API

## pthread_create

POSIX 线程使用 `pthread_create()` 创建线程。调用者提供线程标识保存位置、属性、线程入口函数和传给入口函数的参数。新线程创建后，会在同一进程地址空间中执行入口函数，入口函数参数和返回值都使用 `void *`，因此可以传递任意指针类型。

入口函数形如 `void *(*start_routine)(void *)`。如果需要传多个参数，常把它们打包进结构体，再把结构体指针传入。要注意参数对象的生命周期，不能把即将失效的栈对象地址传给长期运行线程。

## pthread_join

`pthread_join()` 用来等待指定线程结束，并可接收线程返回值。调用线程会阻塞，直到目标线程退出。它类似父进程对 `wait()` 的需求：不仅同步“完成”事件，也负责回收线程终止后的部分资源。

如果不需要返回值，可以传入 `NULL`。如果线程返回指针，该指针必须指向仍然有效的对象。返回线程栈上局部变量地址是错误的，因为线程函数返回后栈帧失效。

## 参数与返回值

线程参数和返回值常见错误来自类型转换和生命周期。整数强转为指针再传递在某些平台上不安全，尤其当整数宽度和指针宽度不同。更稳妥做法是使用结构体，或分配专门参数对象。

返回复杂结果时，可以由调用者预先分配结果结构，也可以由线程在堆上分配并由 join 方释放。接口应明确谁负责释放，避免泄漏或重复释放。

## 线程属性

`pthread_attr_t` 可以设置栈大小、分离状态、调度属性等。多数情况下默认属性够用，但大量线程或深递归程序需要关注栈大小。栈太小会溢出，栈太大会浪费虚拟地址空间和内存承诺。

分离线程不需要被 join，结束后资源自动回收。它适合不关心返回值的后台任务，但一旦分离，就不能再用 join 等待结果。

## 错误处理

线程 API 返回错误码，不一定设置 `errno`。健壮代码应检查返回值。创建线程可能失败，例如资源不足、属性无效或达到线程数量限制。忽略失败会让程序后续等待未创建线程或访问未初始化标识。

## 易错点

创建线程后立即 join，通常等价于同步函数调用，不能发挥并发价值。线程创建顺序也不保证执行顺序，先创建的线程不一定先运行。join 是同步完成，不是保护共享数据；共享数据仍需锁、条件变量或其他同步机制。
